Pipeline · Autonomous Loops · Linear

Every run, one Linear issue. Automatically.

Two autonomous loops, two Linear projects, one poster module. gtm-autoresearch files into the Autoresearch project on every loop completion. AutoAgent harness experiments — wrapped by Hermes on claws-mac-mini — file into their own project. Title encodes the score, body embeds the run notes plus GitHub blob links to four artifacts. Commits referencing the issue key auto-link via Linear's native GitHub integration. Secrets stay on the host.

End-to-end pipeline

Six steps from loop end to triage.

TRIGGER · 01
Loop reaches a stop condition
Plateau (≥92% × 3 rounds), regression cap (3 reverts), JSON-fail cap (5), or max rounds. Final winning container is selected. AutoAgent stops on Harbor target or wallclock budget.
WRITE · 02
Four artifacts to disk
Winning JSON, structured experiment log JSON, human-readable notes MD, data-audit MD. Written to content/gtm-templates/{client}/loop-results/. Source of truth — Linear is downstream.
POST · 03
Linear GraphQL: issueCreate
scripts/post-to-linear.ts reads the notes MD, builds the title from score deltas, prepends artifact blob links, fires issueCreate. Skips silently if LINEAR_API_KEY unset.
LINK · 04
GitHub commits auto-attach
Linear's native GitHub integration watches commits/PRs in Organized-AI/gtm-autoresearch. Any reference to the issue key (e.g. BLU-12) attaches a preview to the issue. No code needed.
REVIEW · 05
Single triage surface
Filter the Autoresearch project by client label or score delta. Comment on regressions. Re-open if a deploy reveals the winning config didn't actually improve live ads metrics.
BOUNDARY · 06
What the poster doesn't do
Doesn't run the loop. Doesn't deploy the GTM container. Doesn't replace the disk artifacts. Doesn't conditional-label by improvement state (yet). Pure post-hoc record.

linear-worker · the reverse path

When Linear needs to talk back.

The poster module handles outbound writes — claws-mac-mini → Linear. The reverse direction (Linear → other systems) flows through linear-worker.jordan-691.workers.dev, a Cloudflare Worker that receives Linear's webhooks and fans them out.

# Probe: GET / returns 405 (POST-only webhook receiver) curl -i https://linear-worker.jordan-691.workers.dev/ → HTTP 405 Method Not Allowed # Health check works curl https://linear-worker.jordan-691.workers.dev/health → {"ok":true}

Configured as the webhook target in Linear → Settings → API → Webhooks, the worker receives events for Issue, Comment, IssueAttachment, etc. on both projects (Autoresearch + AutoAgent). From there it can:

  • Notify Slack when a regression issue gets a comment
  • Trigger Hermes to re-run a loop when an issue is reopened with a specific label
  • Persist to KV for downstream analytics on regression patterns
  • Forward to GitHub as workflow_dispatch when an issue transitions to a "deploy" state

The worker keeps secrets at the edge: webhook signing secret + any downstream API tokens live in Cloudflare Worker Secrets, not on claws-mac-mini. Outbound posting (poster module) and inbound webhooks (worker) form a closed loop where Linear is the coordination plane and neither end of the loop holds the other's credentials.

Why this shape

Linear is downstream of disk, not in front of it.

The autoresearch loop produces four structured artifacts per run — winning config, experiment log, run notes, data audit. Those files are the source of truth: committable, diffable, replayable, survivable if Linear goes down. The Linear post is a render of those artifacts into a triage-friendly form — not a write that anything depends on.

Practically: if LINEAR_API_KEY is unset, the loop runs identically and produces identical disk output. Linear failure modes (rate limit, downtime, key revoke) degrade gracefully to a single console.warn, never fail the loop. This is also why the issue files into the Done state on creation — the work that produced it has already happened, and Linear's role is recordkeeping plus discovery, not workflow gating.

The same logic applies when Hermes wires AutoAgent in Phase 4. AutoAgent's harness/trajectory artifacts live on claws-mac-mini first; the Linear post is the public-facing index into them.
